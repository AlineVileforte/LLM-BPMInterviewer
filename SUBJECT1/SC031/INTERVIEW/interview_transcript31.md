Relatório da Entrevista SBMN
Perguntas e Respostas
1. What is the name of the process?
Resposta: IT Solution Deployment Process
2. What is the domain or business area of this process?
Resposta: Information Technology
3. Please provide the complete list of activities in the process.
Resposta: Submit IT solution request; Assess compatibility; Evaluate cost and resource needs; Procure necessary tools or licenses; Install the solution; Test solution; Roll out solution to requesting department; Provide training; Provide support for troubleshooting
4. After Submit IT solution request, can Assess compatibility occur only after the request has been submitted?
Resposta: YES
5. Are there valid executions of the process where Submit IT solution request occurs but Assess compatibility does not occur?
Resposta: NO
6. Can Evaluate cost and resource needs occur only after Assess compatibility has been completed?
Resposta: NO
7. Can Assess compatibility occur only after Evaluate cost and resource needs has been completed?
Resposta: NO
8. Can Procure necessary tools or licenses occur only after Evaluate cost and resource needs has been completed?
Resposta: YES
9. Are there valid executions where Evaluate cost and resource needs occurs but Procure necessary tools or licenses does not occur?
Resposta: YES
10. Can Install the solution occur only after Procure necessary tools or licenses has been completed?
Resposta: YES
11. Are there valid executions where Procure necessary tools or licenses occurs but Install the solution does not occur?
Resposta: NO
12. Can Test solution occur only after Install the solution has been completed?
Resposta: YES
13. Are there valid executions where Install the solution occurs but Test solution does not occur?
Resposta: NO
14. Can Roll out solution to requesting department occur only after Test solution has been completed?
Resposta: YES
15. Are there valid executions where Test solution occurs but Roll out solution to requesting department does not occur?
Resposta: NO
16. Can Provide training occur only after Roll out solution to requesting department has been completed?
Resposta: YES
17. Are there valid executions where Roll out solution to requesting department occurs but Provide training does not occur?
Resposta: YES
18. Can Provide support for troubleshooting occur only after Provide training has been completed?
Resposta: NO

Resposta: NO

Resposta: YES

Resposta: YES

Resposta: YES

Resposta: YES

Resposta: YES
25. Can Procure necessary tools or licenses occur only after both Assess compatibility and Evaluate cost and resource needs have been completed?
Resposta: YES

Resposta: YES

Resposta: YES

Resposta: YES
29. Can Provide training and Provide support for troubleshooting occur together in the same process execution?
Resposta: YES
30. Are valid executions possible with only Provide training, only Provide support for troubleshooting, and both?
Resposta: YES
31. If Test solution identifies problems or fails, can the process return to an earlier activity for rework or correction?
Resposta: NO
32. If Provide support for troubleshooting identifies an issue, can the process return to an earlier activity for correction or rework?
Resposta: YES, RETURN ITSELF
33. After Provide support for troubleshooting is completed, can Provide support for troubleshooting be executed again?
Resposta: YES
