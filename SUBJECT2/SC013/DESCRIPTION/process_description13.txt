This process represents a logistics scheduling workflow used to manage customer requests for changing a previously defined pick-up time. The process begins when a customer requests a modification to the scheduled pick-up time.

First, the organization asks for both the old and the new requested pick-up times. After receiving the change details, the process verifies whether the script is still pending data entry. Based on this verification, different execution paths are followed.

If the requested change involves a later or earlier pick-up time and the script is still pending data entry, the process updates the pick-up time on the script and moves the script to the new timeslot. After the update is completed, the organization approves the new pick-up time and confirms the new schedule to the customer.

If the request involves a later pick-up time after the data entry phase, the process directly approves the new pick-up time. However, if the customer requests an earlier pick-up time after the script has already passed data entry, the process declines the request and explains that the change is not possible. The workflow concludes either with the approval of the new pick-up time or with maintaining the original schedule.
